## PRODUCT CONTEXT

I am building an Android mobile application called **DocVault**.

It is an **offline-first, AI-powered document organizer** that:

* Scans a user's phone for documents
* Converts them to PDF (if needed)
* Encrypts everything locally
* Extracts text using on-device OCR
* Automatically categorizes documents
* Allows full-text search
* Never sends any data to the cloud

The core philosophy is:

> ‚ÄúYour documents, organized by AI ‚Äî but everything stays on your device.‚Äù

---

## TARGET USERS

* Privacy-conscious users
* People overwhelmed by scattered documents
* Users receiving documents via WhatsApp, Downloads, Camera
* Anyone who wants automatic organization without cloud dependency

1. 100% local processing
2. No cloud sync

## MVP FEATURE SET (V1)

### 1. Document Discovery

* Scan Downloads, Documents, DCIM, WhatsApp
* Scan images, PDFs, Office files
* Duplicate detection via MD5
* Filter out memes and small files

---

### 2. Import Pipeline

When importing a document:

1. Convert image to PDF (if needed)
2. Run OCR on image or scanned PDF
3. Extract metadata (dates, amounts, names, document numbers)
4. Auto-generate a meaningful filename
5. Auto-categorize using keyword rules (and optionally ML)
6. Encrypt file with AES-256
7. Store in encrypted local database
8. Optionally delete original

---

### 3. AI Capabilities

#### OCR

* Extract text from images and PDFs
* Fallback to bitmap rendering if scanned PDF

#### Metadata Extraction

* Regex-based:

  * Dates
  * Currency
  * Names
  * Invoice / policy numbers

#### Title Generation

Rule-based naming:

* Vendor + Type + Date
  Example:
* Amazon_Receipt_Jun2025.pdf
* Passport_JohnDoe.pdf

#### Auto Categorization

Categories:

* ID & Personal
* Financial
* Receipts
* Medical
* Education
* Vehicle
* Property
* Other

Primary method:

* Weighted keyword scoring

Optional method:

* TFLite classifier
* Fallback to keyword model

User corrections are stored for future AI improvement.

---

### 4. Security Model

* Biometric lock
* PIN fallback
* Auto-lock after background timeout
* SQLCipher encrypted database
* AES-256 encrypted files
* Per-file encryption keys stored in Android Keystore
* Encrypted backup ZIP file protected by PIN
* No network dependency

---

### 5. Search

* Full-text search (Room FTS4)
* Search across:

  * Title
  * OCR text
  * Category
  * Metadata
* Results < 200ms

---

### 6. Capture & Share

User can:

* Capture document via camera
* Pick file from system
* Share PDF/image TO DocVault
* Share decrypted PDF FROM DocVault
* Download decrypted copy to storage

---

### 7. Backup & Restore

* Single encrypted ZIP export
* Contains encrypted files + encrypted DB + metadata
* Protected by user PIN
* Fully offline restore
* 30-day backup reminder

---

### 8. Monetization

Freemium model:

* Free: 10 documents

* Unlimited documents
* Google Play Billing (non-consumable)
* Restore purchase option
